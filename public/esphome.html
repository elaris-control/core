📖 ESPHome Guide

Use the Peripheral Library to generate a YAML for a sensor (anemometer, rain, CO2, etc.) and flash it onto a generic ESP32 board.

1
Wire the sensor to an ESP32 dev board. Note the GPIO pin used.
2
Click Peripheral Library → select your sensor type.
3
Fill in: Device Name (e.g. wind-station), GPIO pin, MQTT Broker IP (Pi's IP), WiFi credentials.
4
Click Generate YAML → Copy the YAML to clipboard.
5
Click Import YAML (topbar) → paste the YAML → Import. This saves the sensor as a board profile.
6
In Step 1 of the installer: select the newly imported profile.
7
Connect ESP32 via USB → select the serial port → click Flash.
8
After flash the ESP32 reboots and connects to WiFi + MQTT automatically.
9
See tab 3 · After Flashing for next steps in ELARIS.
💡 Use a data USB cable (not charge-only). Most ESP32 boards use CP2102 or CH340 — install the driver on your PC if the port doesn't appear.

Flash a Kincony relay board (KC868-A4/A8/A16/A32) using the built-in board profiles — all GPIOs and I2C expanders are pre-configured.

1
In Step 1 select your board (e.g. Kincony KC868-A16).
2
Enter a Device Name (e.g. kc868-living) — lowercase, numbers and dashes only.
3
Enter WiFi SSID & Password. For Ethernet boards enable the Ethernet toggle and leave WiFi blank.
4
Enter MQTT Broker — your Pi's local IP (e.g. 192.168.1.10).
5
Click Next to review entities (relays, inputs). Rename them if needed — the names appear in ELARIS.
6
Click Next → YAML preview. Connect board via USB, select port → Flash.
7
Board reboots after flash and connects to MQTT. IOs appear in ELARIS within ~30 seconds.
💡 If ESPHome is not installed, use the Install ESPHome button at the top of this page first.

What to do in ELARIS after a successful flash.

1
The ESP boots and publishes sensors/relays to MQTT. This takes 15–30 seconds.
2
Open ELARIS main app → go to Installer (plug icon). Your device appears with pending IOs.
3
Click Approve on each IO you want to use. Approved IOs become available for automation.
4
Go to Automation → open or create a module instance (e.g. Awning Control).
5
In Map IOs: assign your approved IOs to module inputs. E.g. map wind-station / Wind Speed → Wind Sensor of the Awning module.
6
Set Setpoints (e.g. retract above 40 km/h) → Save. Automation is now live.
💡 You can mix IOs from different devices in one module: relays from KC868-A16 + wind sensor from a separate ESP32 — all via MQTT.
💡 Device not appearing? Check: (1) MQTT broker running, (2) Mosquitto listens on 0.0.0.0:1883 not just localhost, (3) device and Pi on same network.
